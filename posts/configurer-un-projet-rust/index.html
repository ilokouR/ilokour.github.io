1. Le canal de release Rust

3 types de canales de developpement de Rust sont en cours de dev simultanement:

  • Canal stable:

    • Les fonctionnaités et Librairies stables du langages

  • Canal beta:

    • Les fonctionnaités et Librairies instables du langages

  • Canal nightly:

    • Les fonctionnaités et Librairies instables du langages

Rustup:
Est un outil pour installer: Le compilateur de Rust, La librairie standard, Cargo, le package manager et d’autres beaucoup d’autres outils tel que : La Doc, L’outil de formattage de code, etc…

Tous ces outils sont disponible en plusieurs catégories qu’on appel : les toolchains
Toolchain : = Combinaison d’un type de canal de dev (release: stable, beta, nightly) + un host(la plateforme: windows, Linux, macos etc..).

Rustup determine le type de type en fonction de la plateforme : Macos ou Windows ou Linux: Il(rustup) agit comme un multiplexer… un peu comme : pyenv => python ou bien nvm => node.js

La version stable de rust est mise à jour chaque 6 semaines(1 mois 2 semaines) La version nightly de rust est mise à jour chaque jour, et chaque chaque 6 semaines(1 mois 2 semaines) la branch master de la version nightly devient la version beta.

Ceux qui veulent utiliser des fonctionnalités de la version nightly doivent utiliser des annotations dans le code pour activer les fonctionalités(flags) qu’ils ont besoins.

#![feature(try_trait)]

Les versions stables et beta n’utilisent pas l’annotation de feature.

Rustup est configuré pour utiliser la version stable par defaut. Pour changer de version ( Visitez pour plus de commands) :

Pour installer la version nightly :

rustup toolchain install  nightly

Pour activer la version nightly de Rust globalement :

rustup default nightly

Pour activer la version nightly de Rust au niveau d’un dossier seulement :

rustup override set nightly

Pour avoir la version du compilateur, nightly :

rustup run nightly rustc --version

Pour revenir à la version stable de Rust :

rustup default stable

Pour voir la version de toolchains installé et actuellement utilisée:

rustup show

Pour faire la mise à jour de la version de toolchains installé et actuellement utilisée:

rustup update

Quel version de Rust faut’il utiliser ?

  • S’il s’agit d’un projet en production, il faut utiliser la version stable

2. Les types de projets Rust

  • Les librairies ou crates ou lib crate

    • Pour la reutilisation du code
    • L’exécution d’une librairie commence dans le fichier /src/lib.rs

  • Les binaires ou executables

    • Ce sont des programme complet, tout est inclus, prêt à être deploillé
    • L’exécution d’un programme commence dans le fichier /src/main.rs
